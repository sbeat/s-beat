Bedingungen für Studenten

Die Bedingungen für Studenten werden in der Administration unter Definitionen konfiguriert. Auf Basis dieser Bedingungen werden Risikokriterien gefunden.

Die Definition der Bedingungen erfolgt mittels Queries (auch Datenfelder genannt). Ein Query ist ein Attribut eines Studenten. Dazu gehören auch die zuvor für einen Studenten berechneten Werte. Für jedes Query gibt es mehrere Bedingungen. Jede Bedingung deckt dabei einen Wert oder einen Wertebereich ab.

Zu einem Query gehören eine Formatierungsangabe, eine Datengruppe und Abhängigkeiten. Mit der Formatierungsangabe weiß das Frontend wie es den Wert ausgeben soll und wie der Wert gefiltert werden kann. Die Datengruppe bestimmt unter welcher Hierarchie das Query in der Filterauswahl im Frontend zu finden ist. Hierarchie-Ebenen werden dabei mit einem Punkt getrennt.

Für das erste, zweite und dritte Semester eines Studenten werden die nachfolgenden Bedingungen definiert. Diese sind davon abhängig, ob study_time_real größer oder gleich dem geforderten Semester ist. Es würde keinen Sinn machen die aufgeführten Bedingungen bei einem Studenten zu prüfen, der noch kein Semester mit abgeschlossenen Prüfungsleistungen hat.
